> 배경: 플레이테스트 대화 분석을 "NPC 자연스러움·대화 이어짐" 두 축으로 재검토.

## A. 어체 자기모순 교정 (선행)

**증상**: 펠릭스가 T12 통째로 합쇼체(~습니다), T13 하오체+합쇼체 혼용.
**근본**: NPC 정의 안에서 `personality.speechRegister`(HAPSYO) ↔ `speechStyle` 산문(하오체, "금지 ~합니다")이 **정면 모순**. 프롬프트가 둘 다 주입 → LLM 진동. R5v2 교정은 register(HAPSYO) 기준이라 ~습니다를 정상 처리, 하오체 어미는 교정표 밖이라 통과.

**스캔·교정**: 3개 고정 팩 전수(금지목록 제거 + 어체 명칭 우선 판정) → 진성 모순 **3건**(펠릭스·라이라·올드릭, 전부 HAPSYO↔하오체). 산문이 정본이라 field를 HAOCHE로 교정. 정본 `content-validator.inferRegisterFromStyle`가 "금지:" 목록에 취약해 1차 24건 중 21건이 FP였음(검증기 하드닝은 백로그).
**검증**: 펠릭스 T12~15 순수 하오체(합쇼체 누출 0), 로넨(HAPSYO 유지) 무피해.

## B. 자연스러움 3종

대화 분석 판정: **연속성 = 상**(대화 잠금 4턴·화제 계단식 심화·판정 연동 점진 공개), **자연스러움 = 중상**. 깎는 요인 3종 →

### #5 배경 인물 반복 (advance-or-dismiss)
- **증상**: 장면 연속성이 살려두는 무명 감시자("야간 경비 책임자")가 T12~15 매 턴 "무표정하게 훑어본다"만 반복. 발사되지 않는 체호프의 총.
- **근본**: 반복 차단 사전(`BG_IDENTITY_KEYWORDS`)에 계급·간부명 부재 → 탐지 실패. + 장면 연속성(유지) vs 반복 차단(삭제) 규칙이 "정체 감시자 진전"을 못 다룸.
- **수정 (B안)**: auxBlock에 **진전/퇴장 강제 디렉티브** — "계속 지켜보기만 하는 무명 인물(감시자·상급자·책임자)은 이번 턴 반드시 진전(접근·발화·개입·퇴장), 동일 정적 묘사 반복 금지." A안(사전에 계급명 추가 → 삭제)은 긴장 요소 소실이라 미채택.

### #6 제스처 상투구 (L0+L1)
- **증상**: 불안 제스처가 "목덜미를 매만지며/호흡을 가다듬는다/시선 떨구기"에 수렴(cross-NPC).
- **근본 (핵심 학습)**: `recentGestures` 권장 풀이 하필 **과사용 제스처(목덜미·호흡)를 능동 권장**하는 자기강화 루프였고, 추출 whitelist(`GESTURE_PATTERNS`)도 목덜미/호흡을 못 잡음. **정적 어구 풀은 positive/negative 무관하게 anchor**(불변식 41/42, prompt-builder:789 원칙, memory `feedback_concrete_vocab_anchor`). 저모델 LLM은 프롬프트에 넣은 구체 어휘를 반복/변형한다. 하드 whitelist는 무한한 문구를 못 따라가 whack-a-mole.
- **수정**:
  - **L0** — 구체 recommendPool(10개) 삭제 → 동적 "이미 사용" 회피 nudge만 유지.
  - **L1** — 메인 서술 호출에 `frequency_penalty` 0.4 + `presence_penalty` 0.3(미사용이던 모델 레버). `LlmProviderRequest` 필드 + `openai.provider` 양 경로. nano/추출 미설정(결정성), Anthropic/Gemini 미지원 → 무시.
  - **L2(보류)** — 추적 차원을 "제스처 문구"(무한)→"신체 부위"(유한 ~20) 로 바꿔 부위 로테이션 조향. 계측 후 판단.
- **실측**: 최다반복 "목덜미" 0회, 제스처 다양화.

### #7 첫 조우 개방 깊이 티어
- **증상**: 낯선 이에게 3~4턴 만에 목격 비리 고백(과다 개방).
- **근본**: `buildDialogueSeed`(trust 5단계)는 **오프닝 톤만** 게이트, "얼마나 깊은 정보를 스스로 꺼내는가" 상한 부재. 사교 게이트(불변식 44)는 구조화 fact만, 자유 서술식 민감 정보는 무통제.
- **수정**: `em.trust`+`encounterCount` 기반 **개방 티어**를 긍정 프레이밍으로 주입 — 표면(첫 조우 & trust<20: 직업·일상·공개 소문만) / 개인(trust<40: 견해·간접 단서) / 심층(trust≥40: 귓속말 수준). soft 네거티브는 무시되므로 positive framing. 구조화 fact 게이트(서버)는 불변.
- **실측**: 첫 조우 크립틱→단계적 심화(SUCCESS 게이트) 곡선.
