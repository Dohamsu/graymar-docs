# 66. NPC 자기소개 사전 확정 시스템 (2026-07-11)

## 기획

> **때가 되면(성향별 임계 도달) NPC 본인이 직접 이름을 밝힌다.**

기존(arch/64)은 결과(이름 공개)는 3층 방어로 보장했지만, 성사 문면이 사실상 전부
서버 제3자 호명 fallback이었다 — 점검 실측: IntroRollback 43회, 소개 성공은 전부
attempts=2(Fallback), **자기소개 대사 성사 0건**. 7/9 이전 attempts=0 "성공"들은
검증 없이 통과된 조용한 공개(구버전 결함)로 판명.

## 실패의 근본과 접근

메인 LLM에게 "400~700자 서술을 쓰면서 그 안에 자기소개 대사도 넣어라"는 복합
지시는 희석된다(Gemma 별칭 관성 — arch/64 '프롬프트만으론 종결 불가').
→ **LLM 원칙 6**: 좁은 태스크로 분리해 서버가 코드로 검증. 검증된 두 패턴의 조합:
NpcReactionDirector(결정은 nano 먼저, 메인은 표현)와 IntroRollback(사후 검증).

## 3단 보장 사다리

```
임계 도달 (shouldIntroduce — 성향별 1/2/3회, 전 성향 자기소개 통일)
 [1] DialogueGenerator.generateIntroDialogue — nano가 실명 포함 자기소개
     대사 사전 생성. posture별 톤 차등(CALCULATING "재는 듯" 등).
     서버 검증(실명 포함+어체) + 재시도 1회 + 어체별 템플릿 최종 보장
 [2] prompt-builder — [자기소개 — 사전 확정 대사] positive 주입
     ("이 대사를 거의 그대로 포함" — 직전 대사 슬롯과 동일한 검증된 방식)
 [3] llm-worker 5.11 — 성사 판정: 실명이 **따옴표 대사 안**에 등장해야 인정
     (마커 표시명만 실명인 케이스 오판정 방지). 미성사 시 롤백-재시도 없이
     그 턴에 별칭 마커로 사전 대사 삽입 — 지연 0턴, 항상 본인 발화
```

- 재등장 공개(임계 외 특수 공개)와 사전 생성 실패 예외만 기존 외부 경로
  (제3자 호명/단서) fallback 잔존. 엔딩 확정 턴은 소개 비활성 (arch/65 부록 D).
- [IntroDialogueFix]: LLM이 이름 자리에 별칭을 넣은 자기소개("내 이름은
  날카로운 눈매의 회계사이라 하오")는 그 대사의 별칭만 실명으로 교정, 삽입 생략.

## 소개 턴 실명 — 역할 재정렬 (충돌 해소)

새 기획은 "소개 턴 대사 안 실명"을 요구하는데 기존 sanitize는 소개 턴 실명을
별칭으로 되치환 — 정면 충돌로 자기소개 파손·"토토브렌" 텍스트 파괴·이중 소개
회귀 실측. 계층별 단일 책임으로 재정렬:

| 계층 | 담당 |
|------|------|
| 본문·대사 속 실명 | sanitize — **소개 턴부터 허용** (introducedAtTurn ≤ 현재 턴) |
| 마커 표시명 2턴 분리 | getNpcDisplayName(isNameRevealed) + IntroMarkerNorm(실명 마커→별칭) |
| 조용한 노출 방지 | 성사 판정(대사 내 실명) + IntroRollback(예외 경로) |
| 미소개 NPC 실명 차단 | sanitize 유지 (introduced=false) |
