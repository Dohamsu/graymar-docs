# 74. 자율 서사·NPC 생성 — 심층 논의

> 상태: 📎 논의·설계(제안) — 미구현. 2026-07-15 작성.
> 발단: "핵심 NPC와 세계관만 설정해두면 LLM이 스토리와 NPC를 자체 생성했으면
> 좋겠다. 지금 구조상 힘들지?"
> 목적: 이 바람이 현 아키텍처와 어디서·왜 충돌하는지 실측으로 규명하고, 이분법이
> 아닌 스펙트럼으로 실현 가능한 하이브리드를 설계 후보로 정리한다.
> 관련: [[73_scenario_differentiation]](팩 차별화 — 본 논의는 그 대안 축이기도 하다).

---

## 1. 원하는 것 (목표 상태)

> 저작은 **핵심 NPC + 세계관 + 최소 앵커**까지만. 이후 **스토리 전개와 부수 NPC를 LLM이 런타임에 생성**하며, 매 런이 유니크하게 흘러간다.

이것은 "저작된 시나리오 팩"에서 "AI 던전마스터(창발 샌드박스)"로의 이동이다.

## 2. 현 아키텍처의 근본 전제 — 왜 정면충돌하나

핵심 불변식(Critical Design Invariant 1·2):

> **서버가 진실의 원천 · LLM은 서술 전용.** 수치·확률·상태·진행은 전부 서버가 결정론 처리하고, LLM은 텍스트만 만든다. LLM이 실패해도 게임은 진행된다.

이 전제 위에서 모든 하위 시스템이 **콘텐츠로 저작된 구조화 데이터**를 읽는다. 실측으로 확인한 결합 지점:

| 시스템 | 의존 | LLM이 생성하면 깨지는 것 |
|--------|------|--------------------------|
| **NPC 해석** | `NpcResolver.getAllNpcs()` → npcs.json 고정. ContentLoader가 "events가 npcs.json에 없는 NPC 참조 시 경고"까지 함 | 마커·대화잠금·자기소개·fact 게이트·NpcReactionDirector가 전부 `NPC_XXX` id + posture/register/별칭/초상화에 키됨 |
| **진행/엔딩** | 퀘스트 S0→S5는 `discoveredQuestFacts` ↔ `quest.json.stateTransitions` 매칭으로만 전진 | LLM 생성 플롯은 fact가 없어 상태 전이·아크 커밋·엔딩이 발동 안 됨 |
| **일관성** | LLM은 stateless. 구조화 메모리(entity_facts·NpcPersonalMemory·storySummary) 재주입으로 보완 | 자유 생성물은 재주입할 구조 기록이 없으면 표류·모순·증발 (메모리 v2~v4를 만든 그 이유) |
| **표현/에셋** | 마커·초상화·씬 이미지가 콘텐츠 id에 매핑 | 생성 NPC는 초상화·별칭 규약이 없음 |

**결정적 사실 (실측):** 가장 동적인 컴포넌트인 **NanoEventDirector조차 NPC를 "생성"하지 않는다.** `injectedNpcId`는 `content.getNpc(id)`로 검증되고(turns.service:4076), nano 출력의 `npcId`는 주석에 "서버 검증용", `wantNewNpc`는 **미사용 콘텐츠 NPC로 로테이션**할 뿐이다. 즉 전 시스템이 **"저작 풀에서 선택(select)"**이며 **"생성(generate)"은 어디에도 없다.** 이것이 목표와의 근본 간극이다.

## 3. 스펙트럼 프레이밍 — 이분법이 아니다

```
0% ─────────────────────────────────────────── 100%
저작된 테마파크                              AI 던전마스터
(현 위치)                                    (목표 지점)
│                                            │
├ 일관·결정론·테스트 가능                     ├ 무한 콘텐츠·매 런 유니크
├ 아크/엔딩 보장                              ├ 저작 최소
├ 저작 노동 큼 ← 문서 73의 "비슷함"          ├ 표류·아크 미보장·턴당 LLM 비용↑
```

현 시스템은 0%에 가깝고(전부 저작), 목표는 100%에 가깝다(전부 생성). **핵심 통찰: 가치 있는 지점은 양 끝이 아니라 30~50% 하이브리드다.**

### ⚠️ 역설 — 자율화가 "비슷함"을 악화시킬 수 있다 (문서 73 연결)

LLM은 **학습된 기본값 편향**이 있다(CLAUDE.md LLM 설계 원칙 #2). 자유롭게 두면 "수상한 이방인이 다가온다", "안경을 밀어 올리며" 류 제네릭 판타지로 **수렴**한다. 즉 **강한 세계 축·핵심 NPC 앵커링 없이 자율화하면, 문서 73의 "팩이 비슷해진다" 문제가 오히려 심해진다.** 자율성은 다양성을 보장하지 않는다 — 앵커가 다양성을 만든다.

## 4. 이미 깔려 있는 창발 인프라 (생각보다 가깝다)

"LLM 제안 → 서버 구조화 등록" 패턴은 **이벤트·fact·스레드엔 이미 작동**한다:

| 인프라 | 실측 | 성격 |
|--------|------|------|
| **entity_facts** | 실제 DB 테이블 (memories.ts, `(runId, entity, key)` UPSERT + 인덱스) | LLM 출력 → 구조화 영속. **동적 엔티티 저장소의 원형** |
| **WorldFact** | 17파일 | 런타임 세계 사실 구조화 |
| **PlayerThread** | 10파일 | 창발 서사 스레드 추적 |
| **NanoEventDirector / ProceduralEvent** | 8파일 | 매 턴 이벤트 컨셉·선택지 동적 생성 |
| **SituationGenerator / IncidentRouter** | engine/hub | 상황·사건 절차적 생성 |

빠진 건 정확히 두 조각이다:
1. **지속되는 1급 동적 NPC** (현재: 콘텐츠 풀 선택만)
2. **사전 저작 없는 자유 진행** (현재: quest.json S0-S5 필수)

## 5. 제안 아키텍처 — 3층 하이브리드 "디렉터 모드"

핵심 원칙: **LLM은 여전히 "제안"만 하고, 서버가 "구조화 등록"해서 결정론 기계를 계속 돌린다.** 불변식 1·2를 깨지 않고 자율성을 얹는다.

```
┌─ Layer 1: Authored Anchors (저작) ────────────────┐
│  핵심 NPC 소수(CORE) + 세계관 + 앵커 아크 1~2개    │
│  + 세계 축 게이지(문서 73 B1) + 감각 팔레트         │
└───────────────────────────────────────────────────┘
          ↓ 앵커가 생성물을 구속(anchor)
┌─ Layer 2: Dynamic Entity Registry (신규) ─────────┐
│  runtime NPC 등록: NPC_DYN_* + posture/register/   │
│  역할/별칭 stub → runState 또는 dyn_npcs 테이블     │
│  (entity_facts 패턴 재사용). NpcResolver가 콘텐츠   │
│  풀 ∪ 동적 풀을 함께 조회하도록 확장               │
└───────────────────────────────────────────────────┘
          ↓ 등록된 엔티티 위에서
┌─ Layer 3: Emergent Director (신규 승격) ──────────┐
│  PlayerThread + WorldFact를 입력으로 다음 비트 제안 │
│  (SituationGenerator/IncidentRouter를 "빈틈 메우기" │
│  → "이야기 주도"로 승격). fact 없이도 진행하는       │
│  freeform 모드                                     │
└───────────────────────────────────────────────────┘
```

### 5.1 Dynamic NPC Registry (핵심 조각 ①)
- 서술이 새 인물을 요구하면 nano가 **구조화 stub** 생성: `{name, tier: SUB|BG, basePosture, speechRegister, role, unknownAlias, shortAlias, oneLinePersonality}`.
- 서버가 `NPC_DYN_<runId>_<seq>` id 부여 + 영속(runState.dynamicNpcs 또는 신규 테이블).
- `NpcResolver.getAllNpcs()`를 **콘텐츠 ∪ 동적**으로 확장 → 마커·대화잠금·자기소개·반응 디렉터가 **코드 변경 없이** 동적 NPC를 콘텐츠 NPC처럼 취급.
- 초상화: 무명 실루엣 폴백 재사용(이미 존재) 또는 런타임 생성(과금 이슈로 후순위).
- **가드**: 생성 stub를 서버가 검증(enum·어체·별칭 길이 — 이미 ContentValidator/마커 새니타이즈 로직 존재)해 오염 차단.

### 5.2 Emergent Director (핵심 조각 ②)
- 입력: 핵심 NPC 상태 + 세계 축 게이지 + PlayerThread + 최근 WorldFact.
- 출력: 다음 "비트"(장면 씨앗·관여 NPC·선택지·선택적 새 WorldFact).
- quest.json 없이도 도는 **freeform 진행**: S0-S5 대신 thread 진척도/게이지 임계로 클라이맥스·엔딩 트리거.
- 앵커 아크가 있으면 director가 그쪽으로 인력(gravity)을 걸어 표류 방지.

### 5.3 Consistency Harness (자율의 대가 상쇄)
- 모든 생성 엔티티·비트를 entity_facts/WorldFact로 즉시 구조화 → 다음 턴 선별 주입(불변식 24).
- "이 NPC가 이미 한 말/한 행동"을 강하게 재주입해 stateless 표류 억제.

## 6. 자율 수준별 옵션 (의사결정용)

| 레벨 | 범위 | 저작량 | 리스크 | 비고 |
|------|------|--------|--------|------|
| **L0 (현재)** | 전부 저작 | 최대 | 최소 | 문서 73 "비슷함" |
| **L1** | 동적 **배경 NPC**만 생성(5.1), 진행은 저작 유지 | 큼↓ | 낮음 | 마커/초상화 폴백 검증만. **최소 파일럿 후보** |
| **L2** | L1 + 사이드 비트를 director가 생성(5.2 부분) | 중 | 중 | A2 텍스처 이벤트를 런타임 생성으로 대체 |
| **L3** | 핵심 NPC·앵커 아크만 저작, 본류도 창발 | 최소 | 높음 | 표류·아크 미보장·비용↑. freeform 모드 필요 |

→ **A2의 재해석**: 지금 손으로 찍는 텍스처 이벤트 44개는 정확히 **L2가 런타임 생성으로 대체할 층**이다. L1/L2가 서면 A2 저작은 "생성 실패 시 폴백 풀"로 격하된다.

## 7. 리스크·미해결 질문

1. **비용** — 턴당 LLM 호출 증가(디렉터 + NPC 생성 + 일관성 추출). nano 다단이라 레이턴시 10초 미만 전제(메모리)와 충돌 가능. 계측 필요.
2. **표류 vs 앵커 인력의 균형** — director가 앵커로 얼마나 세게 당길지 미검증. 너무 약하면 표류, 너무 세면 결국 L0처럼 획일.
3. **테스트 가능성 급락** — 결정론 팩은 playtest V-체크로 회귀 검증되나, 창발은 "매번 다름"이라 골든 대조가 안 됨. **분포 지표**(문서 73 §8) + 불변식 위반 감지로 전환 필요.
4. **엔딩 보장** — freeform에서 "런이 끝나긴 하는가"(불변식 19 최소 15턴). 게이지/thread 종료 조건 설계 필수.
5. **동적 NPC 초상화** — 실루엣 폴백으로 갈지, 생성 켤지(과금).
6. **"비슷함" 역설(§3)** — 자율화가 목표(다양성)를 배신하지 않는지 §8 지표로 반드시 계측.

## 8. 권장 착수 — 가장 작은 검증부터

큰 방향을 정하기 전에 **L1 동적 배경 NPC 레지스트리 스파이크** 하나로 핵심 가정을 싸게 검증한다:
- nano가 BG NPC stub 1명 생성 → `NPC_DYN_*` 등록 → 마커·대화·기억이 콘텐츠 NPC처럼 도는가?
- 초상화 폴백·별칭 새니타이즈·다음 턴 재등장 일관성이 유지되는가?
- 이 스파이크가 통과하면 5.1은 "코드 변경 없이 NpcResolver 풀 확장" 가설이 검증되고, L2/L3 논의의 토대가 된다.

실패하면 자율화 비용이 예상보다 크다는 신호 — L0/A2(저작) 유지가 합리적.

## 9. 결론 (현시점 판단)

- 목표(완전 자율)는 **불변식 1·2와 정면충돌**하며, 그대로는 어렵다 — **NPC·진행이 저작 데이터에 하드 결합**돼 있기 때문(실측).
- 그러나 **"LLM 제안 → 서버 구조화"의 인프라(entity_facts 테이블·WorldFact·PlayerThread·nano 디렉터)는 이미 절반 존재**한다. 빠진 건 동적 NPC 레지스트리와 freeform 진행 두 조각.
- 올바른 목표는 100%가 아니라 **30~50% 하이브리드(디렉터 모드)**. 불변식을 깨지 않고 "제안-등록" 패턴을 NPC·진행으로 확장하는 것.
- 최대 함정은 **LLM 기본값 편향 역설** — 앵커 없는 자율은 문서 73의 획일화를 악화시킨다. 자율성 자체가 아니라 **강한 세계 축·핵심 NPC 앵커**가 다양성의 원천이다.
- 다음 행동: **L1 스파이크로 싸게 검증** 후 L2/L3 진입 판단. A2(수동 이벤트)는 이 결정 전까지 보류하거나 폴백 풀 관점으로만 최소 진행.
