# 25. LLM 모델 평가 보고서 — GPT-4.1-mini vs Gemma 4

> 2026년 4월 5~6일 실시한 LLM 모델 비교 평가.
> 그레이마르 텍스트 RPG의 내러티브 생성에 최적인 모델을 선정.

## 1. 배경

### 1.1 현재 시스템
- 서버: NestJS, multi-provider LLM 시스템 (openai/claude/gemini/mock)
- 용도: 턴당 한국어 내러티브 서술 생성 (중세 판타지 경어체)
- 프롬프트: 시스템 ~10,000자 + 메모리 ~5,000자 + 유저 ~3,000자
- 출력: 400~700자 한국어 서술

### 1.2 기존 모델
- **gpt-4.1-mini** (OpenAI) — 기존 메인 모델
- 입력: ₩0.60/1K토큰, 출력: ₩2.40/1K토큰 (USD $0.40/$1.60 per M, 환율 ₩1,500)

### 1.3 평가 동기
- Gemma 4 출시 (2026.4.2) — Apache 2.0 오픈 모델
- 비용 절감 + 품질 비교 필요

## 7. Rate Limit 문제

### Gemini API (Google AI Studio)
- Gemma 4 모델은 분당 16K 토큰 제한
- 게임 프롬프트 1회 = ~17K 토큰 → 1분에 1회만 가능
- 유료 티어(Tier 1)에서도 동일 — 오픈 모델이라 자체 모델보다 할당 낮음
- **실서비스 부적합**

### OpenRouter
- 크레딧 기반, TPM 제한 거의 없음
- 연속 호출 5턴 테스트 — rate limit 0회
- 10턴 테스트 — rate limit 0회
- **실서비스 적합**

### OpenAI
- gpt-4.1-mini Tier 1: 2,000,000 TPM
- rate limit 걸린 적 없음
- **실서비스 적합**
