# Run Planner & Node Generation Spec v1.1 (Tone Applied)

## RUN 템포/성향 반영 업데이트 버전

------------------------------------------------------------------------

# 0. RUN 성향 확정

본 버전은 다음 성향을 반영한다:

-   평균 RUN 길이: 9\~12 노드 (긴 몰입형)
-   전투 밀도: 낮음 (EVENT 중심 구조)
-   생존 압박: 중간
-   보스: 확률 등장
-   EXIT: 중반 이후 확률 등장
-   전체 톤: 성장 체감형 RPG

------------------------------------------------------------------------

# 1. RUN 구조 개요

## 1.1 기본 Depth 구조 (권장 예시)

-   Depth 1\~3: 서사 중심 (EVENT 다수)
-   Depth 4\~7: 성장 체감 구간 (COMBAT 점진적 증가)
-   Depth 8\~12: 클라이맥스 구간 (보스 확률 상승 + EXIT 등장 가능)

실제 Depth 구간 수치는 외부 플랫폼에서 확정한다.

------------------------------------------------------------------------

# 2. 노드 분포 정책 (성향 반영)

## 2.1 기본 가중치 (상대적 비율 개념)

-   EVENT: 높음
-   COMBAT: 낮음\~중간
-   REST: 중간
-   SHOP: 낮음\~중간
-   EXIT: 중반 이후 활성화

> 정확한 확률 값은 외부 튜닝 대상

------------------------------------------------------------------------

# 3. Phase별 정책

## PHASE 1 (초반, Depth 1\~3)

-   EVENT 중심
-   COMBAT는 약한 적 위주
-   REST 노출 확률 중간
-   SHOP 낮음

목표: 세계관 적응 + 초기 성장 기반 형성

------------------------------------------------------------------------

## PHASE 2 (중반, Depth 4\~7)

-   EVENT와 COMBAT 균형
-   성장 체감 강화
-   SHOP 등장 확률 증가
-   보스는 아직 낮은 확률

목표: 빌드 방향 체감

------------------------------------------------------------------------

## PHASE 3 (후반, Depth 8\~12)

-   보스 확률 상승
-   EXIT 후보 활성화
-   EVENT는 분기/결말 성격 강화
-   REST는 제한적 유지

목표: 긴장 + 성장 확인 + 결말 유도

------------------------------------------------------------------------

# 4. 보스 정책

-   RUN당 최소 0, 최대 1 보스 등장 (권장)
-   등장 조건:
    -   Depth 7 이상
    -   이전에 보스 등장하지 않았을 것
-   보스는 개별 드랍 롤 적용

------------------------------------------------------------------------

# 5. EXIT 정책

-   Depth 6 이후 확률 활성화
-   보스 등장 이후 EXIT 확률 상승
-   EXIT는 선택형 (플레이어 선택으로 종료)

------------------------------------------------------------------------

# 6. 생존 압박 정책

-   REST는 RUN당 최대 2회 권장
-   연속 REST 금지 유지
-   HP 30% 이하 시 REST 가중치 증가
-   전투 밀도 낮기 때문에 회복 기회는 중간 수준 유지

------------------------------------------------------------------------

# 7. 성장 체감 설계 반영

-   COMBAT는 수는 적지만 "질적으로 중요"
-   EVENT에서 능력 성장/평판 상승 기회 다수 제공
-   후반부에 성장 체감이 극대화되도록 보스 배치

------------------------------------------------------------------------

# 8. 테스트 체크리스트

-   평균 RUN이 9\~12 노드 내에서 종료되는지 확인
-   EVENT가 전체 노드의 절반 이상인지 확인
-   보스가 과도하게 자주 등장하지 않는지 확인
-   EXIT가 너무 일찍 등장하지 않는지 확인
-   REST가 과도하게 많아지지 않는지 확인

------------------------------------------------------------------------

# 결론

v1.1은 긴 몰입형, EVENT 중심, 성장 체감형 RPG 구조를 반영한 플래너
스펙이다. 세부 가중치 및 수치는 외부 플랫폼에서 밸런싱한다.
