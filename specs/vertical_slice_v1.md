# Vertical Slice Spec v1 (Canonical)
## 1회 완주 콘텐츠 설계 정본 (실종 사건 추적 편)

---

# 0. 슬라이스 목적

본 Vertical Slice의 1차 목적은 **서사 검증(B)** 이다.

- LLM 기반 스토리 일관성 확인
- NPC 관계/전술 성향 반영 확인
- 중후반 긴장 구조 및 엔딩 분기 테스트

시스템 검증은 부차적이며, 핵심은 “몰입 가능한 한 판” 완성이다.

---

# 1. 메인 테마

## 실종 사건 추적

플레이어는 항만 도시에서 발생한 실종 사건을 조사한다.
사건은 단순 납치가 아닌, 세력 간 이해관계와 얽혀 있다.

---

# 2. 시작 구조

## 시작 위치: 항만 도시

- 음울한 부두
- 상인 길드
- 경비대
- 실종자의 가족

초반은 조사 중심 EVENT 위주로 구성한다.

---

# 3. RUN 길이

- 10~12 노드 랜덤 구성
- 평균 11 노드 목표
- EVENT 중심 구조 유지

---

# 4. 보스 구조

- 보스는 확률 등장
- Depth 8 이상에서만 활성화
- 보스는 인간형 (항만과 관련된 핵심 인물)

보스 미등장 시:
- 진실 폭로형 엔딩으로 대체

---

# 5. 노드 구성 예시 흐름

## Depth 1
EVENT — 실종 의뢰 수락

## Depth 2
EVENT — 항만 상인과 대화

## Depth 3
COMBAT — 부두 깡패와 충돌

## Depth 4
EVENT — 협상 시도 (중반 핵심: 협상 선택)

## Depth 5
SHOP or REST

## Depth 6
EVENT — 단서 발견

## Depth 7
COMBAT — 길드 사병과 충돌

## Depth 8
EVENT — 진실 일부 공개

## Depth 9~11
보스 확률 등장 or 진실 대면

## Depth 12
EXIT (결말 확정)

---

# 6. 초반 선택 영향

- 첫 선택은 단기 영향만 발생
- 골드/평판/초기 관계 변화
- 후반 강제 재등장 없음

---

# 7. 중반 핵심 구조

중반 핵심 이벤트: 협상

플레이어 선택에 따라:

- 길드와 협력
- 경비대와 협력
- 독자 조사

이 선택은 후반 보스 등장 확률에 영향.

---

# 8. 성장 체감 구조 (복합)

성장 체감은 다음 요소 복합:

- 전투 스타일 누적
- NPC 관계 변화
- 아이템 빌드 형성

후반부에서 플레이어 전술 성향이 서술에 반영되어야 한다.

---

# 9. 클라이맥스

## 인간형 보스

- 길드 간부
- 부패한 경비대장
- 조력자로 위장한 인물

보스는 플레이어 선택에 따라 대사/동기 변화 가능.

---

# 10. 엔딩 구조

2~3개 엔딩 존재

예:

1) 진실 공개 엔딩
2) 타협 엔딩
3) 은폐 엔딩

엔딩은:

- 중반 협상 선택
- NPC 관계 상태
- 보스 처치 여부

에 의해 결정된다.

---

# 11. 분위기 곡선

긴장 유지 → 후반 폭발

- 초반: 조사/의심
- 중반: 갈등 고조
- 후반: 대면/폭로

Tone State는 중간 강도로 유지.

---

# 12. 서술 밀도

몰입형 (길게)

- 장면 묘사 강화
- 감정선 강조
- 대사 중심 구성

단, 토큰 상한은 Memory Spec 기준 준수.

---

# 13. 테스트 체크리스트

- NPC 관계 변화가 후반 엔딩에 반영되는지
- 전술 성향이 보스 대사에 반영되는지
- 협상 선택이 보스 등장 확률에 영향 주는지
- 10~12 노드 내 자연스럽게 종료되는지
- 엔딩 2~3개 정상 분기되는지

---

# 결론

본 Vertical Slice는

“항만 도시 실종 사건”을 중심으로 한
몰입형 서사 검증용 1회 완주 설계이다.

구체적인 이벤트 텍스트 및 적 세트 구성은 외부 플랫폼에서 확정한다.
