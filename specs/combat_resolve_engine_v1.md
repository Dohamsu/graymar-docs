## 전투 수치 엔진 설계 문서

------------------------------------------------------------------------

# 1. 전투 스탯 구조

최종 계산 대상 스탯:

-   ATK
-   DEF
-   ACC
-   EVA
-   CRIT
-   CRIT_DMG
-   RESIST
-   SPEED
-   DAMAGE_MULT (기본값 1.0)
-   HIT_MULT (기본값 1.0)
-   TAKEN_DMG_MULT (기본값 1.0)

배율 스탯 기본값은 모두 1.0이다.

------------------------------------------------------------------------

# 2. 스탯 적용 파이프라인 (Priority 구조)

단계별 순차 적용 방식:

-   100 BASE
-   200 GEAR
-   300 BUFF
-   400 DEBUFF
-   900 FORCED
-   950 ENV

Modifier는 priority 순서대로 적용한다.

------------------------------------------------------------------------

# 3. 명중 공식

    roll + (ACC * HIT_MULT) >= 10 + targetEVA

-   roll = d20
-   1 = 자동 실패
-   20 = 자동 성공
-   치명타는 hit일 때만 판정

------------------------------------------------------------------------

# 4. 치명타 처리

-   치명타 시 DEF 30% 무시
-   CRIT_DMG 배율 적용

# 5. 피해 계산 순서

    baseDamage = ATK * (100 / (100 + effectiveDEF))

    dmg = baseDamage
    dmg *= random(0.9 ~ 1.1)

    if (isCrit)
      dmg *= CRIT_DMG

    dmg *= DAMAGE_MULT
    dmg *= TAKEN_DMG_MULT

    dmg = Math.floor(dmg)

    if (hit && dmg < 1)
      dmg = 1

-   최종 단계에서만 Math.floor 적용
-   적중 시 최소 피해 1 보장

------------------------------------------------------------------------

# 6. 난수 소비 정책

-   hitRoll 항상 소비
-   critRoll, varianceRoll은 hit일 때만 소비
-   RNG state는 turn 단위로 저장

------------------------------------------------------------------------

# 7. 위치 시스템

## SIDE

공격 시: - target.DEF -10%

방어 시: - target.TAKEN_DMG_MULT +10%

## BACK

공격 시: - target.DEF -20% - attacker.CRIT +10%

방어 시: - target.TAKEN_DMG_MULT +25%

-   BACK은 공격 1회만 적용
-   턴 종료 시 FRONT로 복귀

------------------------------------------------------------------------

# 8. BACK 방지 로직

-   FRONT → SIDE → BACK 단계 필요
-   거리 조건 필요 (CLOSE/ENGAGED)
-   1턴 1회 시도
-   실패 시:
    -   stamina -1
    -   angle FRONT 고정
    -   거리 압박

AI는 BACK 시도 시 -3 보정 적용

------------------------------------------------------------------------

# 9. 행동 시스템

## MOVE

-   거리 1단계 변경
-   FRONT → SIDE 시도 가능 (판정 필요)
-   ENGAGED 이탈 시 판정 필요
-   실패 시 stamina -1 + 압박

## EVADE

-   성공 시:
    -   근접 공격 1회 자동 miss
    -   원거리 ACC -5
    -   SIDE 시도 가능
-   실패 시 stamina -1 + 압박
-   효과는 다음 적 공격 1회만 적용

> **v1 구현 현황**: EVADE는 이벤트 생성만 수행. 위 기계적 효과(자동 miss, ACC -5, SIDE 시도)는 미구현 — v2에서 추가 예정.

## DEFEND

-   이번 턴 받는 피해 -30%
-   stamina +1 회복

> **v1 구현 현황**: stamina +1 회복만 구현. 피해 -30% 감소는 미구현 — v2에서 추가 예정.

## FLEE

-   성공 시 전투 즉시 종료
-   전리품 없음
-   경험치 없음

> **v1 구현 현황**: FLEE 스태미나 비용은 1 (일반 Action과 동일). `combat_system.md` §9의 2 스태미나 설계안은 v2 밸런싱 시 재검토.
