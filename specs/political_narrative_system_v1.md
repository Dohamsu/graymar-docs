# Political Narrative System v1
> 통합 설계서: 정치 세력 구조 + 동적 허브 시스템 + 메인 스토리 아크
> 구조: 왕실 + 기사단 + 상단 + 귀족 + 지하세력 | 허브 중심 진행 | 단일 메인 아크 수렴

---

## 1. 세계 구조 개요

현재 왕국은 표면상 안정되어 있다.
그러나 실질적으로는 다섯 세력이 균형을 유지하고 있을 뿐이다.

균형이 무너지면 내전 또는 쿠데타로 이어질 수 있다.

게임에는 하나의 중심 진실이 존재한다.
모든 RUN은 그 진실에 점점 가까워지는 과정이다.
허브는 정보와 정치가 얽히는 공간이다.
플레이어는 점차 중심부로 들어간다.

---

## 2. 세력 구성

### 2.1 왕실 (Royal Court)

- 명목상 최고 권력
- 군권 일부 보유
- 외교/정통성 상징
- 내부 파벌 존재

약점:
- 재정 의존
- 귀족 영향력

### 2.2 기사단 (Order of Knights)

- 군사력 보유
- 국경 방위 담당
- 명예/질서 중시
- 일부 강경파 존재

약점:
- 정치력 약함
- 왕실과의 긴장 관계

### 2.3 상단 (Merchant Consortium)

- 경제력 장악
- 세금/군수 지원
- 외부 무역 네트워크

약점:
- 군사력 없음
- 여론 취약

### 2.4 귀족 연합 (Noble Houses)

- 지방 통치권
- 사병 보유
- 왕실 견제 세력

약점:
- 내부 경쟁
- 명분 부족

### 2.5 지하세력 (Shadow Syndicate)

- 정보/암살/밀수
- 정치적 영향력은 비공식
- 세력 간 중개자 역할

약점:
- 공식 권력 없음
- 노출 시 몰락

---

## 3. 플레이어의 위치

플레이어는:

- 공식적으로는 용병/조사관
- 실제로는 균형을 흔드는 변수

플레이어는 한 세력에 고정 소속되지 않음

---

## 4. 세력 평판 시스템 (Faction Reputation)

대상 세력:

- 왕실
- 기사단
- 상단 (길드 포함)
- 귀족 연합
- 지하세력 (범죄조직)

faction_reputation:
- -100 ~ +100

평판에 따라 변화하는 요소:

- 특별 RUN 해금
- 장비 품질 상승
- 정보 제공 범위
- 암살 대상 지정
- 정치 이벤트 발생

평판 변화 원인:

- RUN 결과
- NPC와의 상호작용
- 정치적 선택
- 특정 사건 트리거

---

## 5. 균형 붕괴 모델

정치 긴장도는 세력 간 갈등으로 상승한다.

붕괴 조건 예:

- 특정 세력 평판 과도하게 상승
- 특정 세력 몰락
- 암살 성공
- 대형 경제 사건

---

## 6. 정치 긴장도 시스템 (Political Tension Level)

political_tension_level (통합 5단계):

| 단계 | 명칭 | 허브 영향 |
|------|------|-----------|
| 1 | STABLE (LOW) | 허브 안전, 일반 RUN 운영 |
| 2 | UNSTABLE (MED) | 소문 증가, 일부 NPC 긴장 |
| 3 | FRACTURED (HIGH) | 허브 안전도 저하, 암살/폭동 확률 상승 |
| 4 | PRE-WAR (CRISIS) | RUN 난이도 급상승, 대규모 이벤트 발생 |
| 5 | CIVIL_WAR | 전면 내전, 허브 기능 제한 |

긴장도에 따른 변화:

- 허브 안전도 변화
- 암살/폭동 확률
- RUN 난이도 변화
- 정치 이벤트 트리거

---

## 7. 허브의 역할

허브는 단순 세이브 공간이 아니다.

허브는 다음의 중심 공간이다:

- 다음 RUN 선택
- 영구 성장
- NPC 관계 변화
- 세력 평판 관리
- 스토리 사건 발생
- 정보 수집
- 장비 배급
- 정치적 선택
- 세력 대표와 대화
- 지원 요청
- 정보 교환

## 9. NPC 관계 시스템

각 NPC는:

- relation_score (-100 ~ +100)
- trust_level (0~5)
- hidden_flags (비밀/과거/배신 가능성)

관계 변화 원인:

- RUN 결과
- DOWNED 횟수
- 특정 선택
- 설득 성공/실패

---

## 10. 배신/전환 가능성

NPC는:

- 세력 전환 가능
- 배신 가능
- 이중 스파이 가능

모두 Story Memory L0에 저장

---

## 11. 허브 이벤트 시스템

허브는 정적 공간이 아니다.

이벤트 예시:

- 암살 시도
- 쿠데타 조짐
- 전염병 발생
- 길드 내부 갈등
- 실종 사건
- 용병 반란

이벤트는:

- RUN 결과에 영향
- NPC 관계에 영향
- 정치 긴장도에 영향

---

## 12. RUN 선택 시스템

허브에서 해금되는 RUN:

- 공개 RUN
- 비밀 RUN
- 고위험 RUN
- 세력 전용 RUN

RUN은 스토리 브랜치 역할을 한다.

---

## 13. RUN의 스토리 역할

각 RUN은:

- 단서 발견
- 세력 변화
- NPC 진실 노출
- 세계관 확장
- 메인 아크 포인트 축적

RUN은 항상 다음 중 하나를 포함:

- 메인 아크 직접 단서
- 간접 단서
- 세력 변화
- 정치적 압박 증가

---

## 14. 허브에서의 자유 입력

허브는 자유 입력 기반이다.

예:

- "대장에게 이번 임무에 대해 묻는다"
- "상단과 거래 조건을 흥정한다"
- "기사단에 지원을 요청한다"
- "마을 소문을 조사한다"

모두 Intent 시스템으로 처리

## 16. 메인 아크 구조 (ACT Structure)

스토리 진행은 정치 긴장도와 연동된다.

| ACT | 정치 단계 | 스토리 단계 | 설명 |
|-----|-----------|-------------|------|
| ACT 1 | STABLE | 이상 징후 / 균열 | 세계 탐색, 초기 단서 발견, 미세한 균열 감지 |
| ACT 2 | UNSTABLE | 배후의 그림자 / 의심 | 배후 세력 조짐, NPC 의심 대상 부상 |
| ACT 3 | FRACTURED | 내부 균열 / 암투 | 세력 간 은밀한 갈등, 비밀 공작 |
| ACT 4 | PRE-WAR | 진실의 단서 / 공개 갈등 | 핵심 진실 접근, 갈등 표면화 |
| ACT 5 | PRE-WAR~CIVIL_WAR | 충돌 / 붕괴 직전 | 전면 충돌 또는 최후 선택 |
| ACT 6 | CIVIL_WAR 또는 해결 | 결말 / 재편 또는 전쟁 | 최종 결말, 세계 재편 |

---

## 17. 메인 아크 진행 방식

메인 아크는 다음 조건으로 다음 ACT로 이동:

- 특정 clue_points 누적
- 특정 사건 트리거
- 특정 NPC 관계 도달
- 특정 RUN 성공

허브에서의 연결:

- 새로운 단서 공개
- 의심 대상 증가
- 정치 긴장도 상승
- 특정 NPC가 의심 대상화
