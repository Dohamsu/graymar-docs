> 핵심: "한 명의 무명 용병이 왕국의 정치 균형을 뒤흔드는 존재로 성장하는 이야기"

# 1. 왕국 구조 & 긴장 확산

## 1.1 왕국 구조

왕국 = 수도(정치 중심) + 귀족 영지 3~5개 + 국경 요새 + 상업/농업 지역

- 표면상 안정, 실질적으로는 5세력이 균형 유지
- 균형 붕괴 시 내전 또는 쿠데타 가능
- 게임 전체에 하나의 중심 진실이 존재하며, 모든 RUN은 그 진실에 접근하는 과정

허브 위치 (ACT 진행에 따라 이동):
- 초기: 수도 인근 용병 집합소
- 중반 이후: 수도 내부 / 귀족 영지 / 국경 군영

## 1.2 RUN 유형

| 유형 | 임무 예시 |
|------|-----------|
| 수도 RUN | 암살 방지, 정보 수집, 귀족 협상 |
| 지방 RUN | 반란 진압, 세력 분쟁, 경제 분쟁 |
| 국경 RUN | 기사단 지원, 외부 위협 조사, 밀수 차단 |

RUN = 에피소드/임무/성장 단위. 주인공은 동일 인물.

## 1.3 정치 긴장 확산

정치 긴장도 `political_tension_level` (5단계):

| 단계 | 명칭 | 영향 |
|------|------|------|
| 1 | STABLE | 허브 안전, 일반 RUN |
| 2 | UNSTABLE | 소문 증가, NPC 긴장 |
| 3 | FRACTURED | 안전도 저하, 암살/폭동 확률 상승 |
| 4 | PRE-WAR | 난이도 급상승, 대규모 이벤트 |
| 5 | CIVIL_WAR | 전면 내전, 허브 기능 제한 |

붕괴 조건: 특정 세력 평판 과잉 / 세력 몰락 / 암살 성공 / 대형 경제 사건

# 2. 주인공 정체성 & 캐릭터 구조

## 2.1 단일 주인공 철학

- 플레이어는 한 명의 인물, 세계는 이 인물을 중심으로 변화
- NPC 기억, 세력 관계, 정치 균형은 **누적** (리셋 없음)
- RUN은 에피소드 단위이며, 세계 상태는 영속

## 2.2 용병 출신 배경

초기 상태:
- 이름 없는 용병, 혈통/정치력/세력 소속 없음, 신뢰도 0
- 기본 전투 능력, 전술 숙련 낮음, 보너스 슬롯 조건 엄격
- 용병 집합소에서 길드 등록, 배급 장비 수령, 낮은 등급 임무만 수락

성장 후:
- 전술 숙련 상승, 보너스 슬롯 활용 증가, 정치적 설득력 상승
- 공식적으로는 용병/조사관, 실제로는 균형을 흔드는 변수

## 2.3 캐릭터 구조 (player_profile)

```
player_profile:
  permanent_stats
  unlocked_traits
  faction_reputation: Record<string, number>   # 세력별 평판
  npc_relations                                 # 대부분 중립
  magic_access_flags
  story_progress
  chapter_index
  political_influence_level
```

# 3. 세력 구조 & 평판 시스템

## 3.1 왕국 5대 세력 (설계 원형)

| 세력 | 역할 | 약점 |
|------|------|------|
| **왕실** (Royal Court) | 명목상 최고 권력, 외교/정통성 상징 | 재정 의존, 귀족 영향력 |
| **기사단** (Order of Knights) | 군사력, 국경 방위, 명예/질서 | 정치력 약함, 왕실과 긴장 |
| **상단** (Merchant Consortium) | 경제력, 세금/군수, 외부 무역 | 군사력 없음, 여론 취약 |
| **귀족 연합** (Noble Houses) | 지방 통치권, 사병 보유 | 내부 경쟁, 명분 부족 |
| **지하세력** (Shadow Syndicate) | 정보/암살/밀수, 비공식 중개자 | 공식 권력 없음, 노출 시 몰락 |

## 3.2 세력 평판 시스템

```
faction_reputation: -100 ~ +100
```

평판에 따른 변화: 특별 RUN 해금, 장비 품질, 정보 범위, 암살 대상 지정, 정치 이벤트 발생
평판 변화 원인: RUN 결과, NPC 상호작용, 정치적 선택, 사건 트리거

## 3.3 현재 구현: 그레이마르 항구 3세력

v1 시나리오(Graymar Harbor)에서는 왕국 5대 세력 대신 **로컬 3세력**을 사용한다.

| 세력 ID | 이름 | 설명 |
|---------|------|------|
| `CITY_GUARD` | 도시 경비대 | 그레이마르 치안 유지, 법 집행 |
| `MERCHANT_CONSORTIUM` | 상인 연합 | 항구 상권 장악, 무역/재화 통제 |
| `LABOR_GUILD` | 항만 노동 길드 | 항만 노동자 조직, 하역/운송 실권 |

**WorldState 저장 구조:**
```typescript
// WorldState.reputation
reputation: Record<string, number>
// 예: { CITY_GUARD: 5, MERCHANT_CONSORTIUM: -2, LABOR_GUILD: 0 }
```

**이벤트 태그 기반 평판 변동:**

| ResolveOutcome | 평판 변동 |
|----------------|-----------|
| SUCCESS | +3 |
| PARTIAL | 0 |
| FAIL | -2 |

- 이벤트의 `factionTag`로 대상 세력 결정
- 한 행동이 하나의 세력에만 영향 (다중 세력 동시 변동 없음)
- Heat, 이벤트 조건 게이트 등에서 평판 참조 가능

## 3.4 균형 붕괴 모델

세력 간 갈등이 정치 긴장도를 상승시킨다.
붕괴 조건: 특정 세력 평판 과잉/몰락, 암살, 대형 경제 사건

# 4. HUB 시스템 & NPC 관계

## 4.1 허브의 역할

허브는 단순 세이브 공간이 아니라 게임 순환의 중심:
- 다음 LOCATION 선택 / 영구 성장 / NPC 관계 변화
- 세력 평판 관리 / 정보 수집 / 장비 배급 / 정치적 선택

## 4.2 현재 HUB 구현

**순환 구조:**
```
HUB → 4 LOCATION (시장/경비대/항만/빈민가) ⇄ COMBAT → HUB
```

**Action-First 파이프라인:**
1. 플레이어가 LOCATION에서 자유 텍스트 입력 (ACTION)
2. IntentParserV2가 ActionType 파싱 (INVESTIGATE, PERSUADE, SNEAK, FIGHT 등)
3. EventMatcher가 6단계 필터링으로 이벤트 매칭
4. ResolveService가 판정 (SUCCESS / PARTIAL / FAIL)
5. 서버가 결과 확정 → LLM이 내러티브 텍스트 비동기 생성

**Heat 시스템 (위험도):**

| Heat 범위 | Safety 등급 | 설명 |
|-----------|-------------|------|
| 0~39 | SAFE | 일반 탐색 가능 |
| 40~69 | ALERT | 경비 강화, 일부 행동 제한 |
| 70~100 | DANGER | 전투 확률 급상승, 위험 이벤트 |

- 한 턴 Heat 변동: **최대 +/-8 클램프**
- Heat 해결: CONTACT_ALLY / PAY_COST 등
- Heat 감쇠: 시간 경과 시 자연 감소

**자유 입력 예시:**
- "대장에게 이번 임무에 대해 묻는다"
- "상단과 거래 조건을 흥정한다"
- "마을 소문을 조사한다"

## 4.3 NPC 관계 시스템

각 NPC:
- `relation_score` (-100 ~ +100), `trust_level` (0~5)
- `hidden_flags` (비밀/과거/배신 가능성)
- 세력 전환, 배신, 이중 스파이 가능 (Story Memory L0 저장)

관계 변화 원인: RUN 결과, DOWNED 횟수, 특정 선택, 설득 성공/실패

## 4.4 허브 이벤트

허브 이벤트 예시: 암살 시도, 쿠데타 조짐, 전염병, 길드 갈등, 실종 사건, 용병 반란
- RUN 결과, NPC 관계, 정치 긴장도에 영향

## 4.5 RUN 선택 & 스토리 역할

- 공개/비밀/고위험/세력 전용 RUN이 허브에서 해금
- 각 RUN은 반드시 포함: 메인 아크 직접/간접 단서, 세력 변화, 정치적 압박 증가 중 하나 이상
- 성장 시스템 상세: -> `04_character_growth.md`

# 5. 메인 아크 ACT 1-6 & 엔딩

## 5.1 영웅 성장 궤적

무명 용병 → 인정받는 용병 → 신뢰받는 중재자 → 정치 변수 → 권력 핵심 → 새로운 질서의 창조자

## 5.2 ACT 구조

| ACT | 긴장 단계 | 영웅 위상 | 핵심 내용 |
|-----|-----------|-----------|-----------|
| 1 | STABLE | 무명 용병 | 세계 탐색, 초기 단서 발견, 미세한 균열 감지 |
| 2 | UNSTABLE | 인정받는 용병 | 배후 세력 조짐, 정보 접근 가능, 정치 개입 시작 |
| 3 | FRACTURED | 신뢰받는 중재자 | 세력 간 은밀한 갈등, 정치적 영향력 확보, 중재자 역할 |
| 4 | PRE-WAR | 정치 변수 | 핵심 진실 접근, 갈등 표면화, 세력 균형 촉매 |
| 5 | PRE-WAR~CIVIL_WAR | 권력 핵심 | 전면 충돌 또는 최후 선택, 세력 균형 좌우 |
| 6 | CIVIL_WAR 또는 해결 | 질서의 창조자 | 최종 결말, 세계 재편 |

## 5.3 ACT 진행 조건

다음 ACT 이동 조건:
- `clue_points` 누적 / 특정 사건 트리거 / NPC 관계 도달 / 특정 RUN 성공

```
story_state:
  act_level: 1~6
  clue_points
  suspicion_flags[]
  revealed_truths[]
  hidden_truths[]
  betrayal_flags[]
  end_condition_progress
```

## 5.4 엔딩 구조

엔딩은 단순 승패가 아닌 질문에 대한 답:
- 어떤 질서를 선택했는가 / 누구를 희생했는가 / 어떤 세력이 남았는가

분기 엔딩:
- **TRUE END** -- 진실 규명 + 세력 재편
- **HARD END** -- 강경 해결
- **POLITICAL END** -- 정치적 타협
- **SACRIFICE END** -- 희생 기반 결말

# 6. 마법 시스템 (간략)

> 현재 미구현. 설계 원칙만 기술.

## 6.1 핵심 원칙

- 마법은 존재하지만 **극히 제한적**이며, 정치 권력과 직접 연결된 자원
- 일반 민중은 마법을 거의 보지 못함 (소문/미신 수준)
- 마법 보유 세력: 왕실(의식/혈통), 기사단(전투 강화/축복), 귀족 일부(고대 유물)
- 마법은 정치적 도구: 정통성 상징, 쿠데타 도구, 암살 수단

## 6.2 플레이어 마법 해금

- 초기: 마법 없음, 순수 전술/기량 기반
- 중반: 세력 연계(왕실 계약/기사단 성물/귀족 동맹/고대 유물) 시 제한적 접근, 대가 수반
- 후반: 제한적 초자연 개입 가능

해금은 "능력 각성"이 아니라 **"정치적 권한 획득"**에 가깝다.

## 6.3 MAGIC_ACTION 카테고리

| Action | 설명 |
|--------|------|
| `ARCANE_STRIKE` | 마법 기반 공격 |
| `WARD` | 마법 방어/보호 |
| `SEAL` | 봉인 |
| `BIND` | 구속/속박 |
| `REVEAL_TRUTH` | 진실 간파 |
| `ROYAL_DECREE` | 왕실 칙령 (스토리 특수) |

## 6.4 제약

- 높은 stamina/별도 자원 소모, 연속 사용 불가, 쿨타임 존재
- 한 RUN당 사용 횟수 제한, 특정 세력 평판 필요
- 남용 시: 페널티 이벤트, 세력 의심도 증가, 적대 관계 발생 가능

# 7. 서사 톤 가이드라인

## 7.1 기본 톤

- 정치 갈등 존재하지만 절망 일변도 아님
- 플레이어는 단순 생존자가 아니라 **"변수"**
- 비극 가능성 있으나 무력감 중심은 아님
- 어둡지만 신비로운 요소 (마법 관련)

## 7.2 감정 구조

| 구간 | 플레이어 감정 |
|------|---------------|
| 초기 | 의심, 탐색 |
| 중반 | 책임감, 긴장감 |
| 후반 | 결단, 희생, 명예 |

## 7.3 NPC 설계 방향

- 각 세력은 나름의 명분 보유, 선/악 이분법 지양
- 대화로 설득 가능, 배신은 개인적 동기 기반

## 7.4 전투/허브 톤

- 전투: 단순 살육 아닌 정치적 의미의 충돌 (암살/구출/저지 임무)
- 허브: 정치의 중심, 음모와 협상의 공간, 영웅의 선택이 기록되는 장소

## 7.5 LLM 서사 가이드

- 과장된 먼치킨/판타지 묘사 지양
- 감정과 정치적 맥락 강조
- 승리에도 비용, 패배에도 명예 유지
- 마법은 희귀하게, 권력과 연결하여 묘사

> Story Memory L0 규칙 정본: -> `05_llm_context.md`
> 메인 진실 관련 사실은 절대 제거 금지 (중요도 0.9 이상 유지)
