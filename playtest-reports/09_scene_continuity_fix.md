## 변경 사항

### 개선 1: sceneFrame 영향력 축소 (3곳 → 1곳)

**문제**: sceneFrame이 LLM 컨텍스트에 3곳에서 중복 전달되어, EventMatcher가 다른 이벤트를 선택하면 LLM이 장면을 점프함.

| 위치 | 변경 전 | 변경 후 |
|------|---------|---------|
| `summary.short` | `[상황] {sceneFrame} [행동] ...` | `플레이어가 ...` (sceneFrame 제거) |
| event text | sceneFrame 전문 | `{actionLabel} — {eventType}` (행동 요약만) |
| `[배경 상황]` 블록 | `[배경 상황]` + 연속성 경고 | `[참고 배경]`으로 명칭 격하 + "어울리지 않으면 완전히 무시" 지시 강화 |

**수정 파일**: `turns.service.ts` (summary.short, event text), `prompt-builder.service.ts` (블록명 변경)

### 개선 2: `[현재 장면 상태]` 컨텍스트 블록 추가

**문제**: LLM이 "누구와 대화 중인지", "어디에 있는지" 명시적으로 알 수 없어 장면 점프 발생.

**해결**: `context-builder.service.ts`에서 현재 장면 상태를 자동 구축:

```
[현재 장면 상태]
아래는 지금 진행 중인 장면의 핵심 맥락입니다. 서술은 반드시 이 장면에서 이어져야 합니다.

대화/상호작용 상대: 후드를 깊이 쓴 정보상 (CAUTIOUS)
장면 배경: 항만의 낡은 선술집에서 정보상이 가죽 가방을 열어 보인다
현재 위치: 항만 부두
이번 방문 3턴째
직전 행동: "가죽 가방 안의 희귀 물건을 조심스레 살펴본다" → 성공
```

**데이터 출처**:
- `actionContext.primaryNpcId` + `getNpcDisplayName()` → 대화 상대
- `actionContext.eventSceneFrame` → 장면 배경 (이번/직전 턴)
- `worldState.currentLocationId` → 현재 위치
- `locationSessionTurns` → 방문 턴 수, 직전 행동

**수정 파일**: `context-builder.service.ts` (currentSceneContext 필드 추가), `prompt-builder.service.ts` ([현재 장면 상태] 블록 삽입)

## 결론

1. **sceneFrame 중복 제거**: summary.short에서 `[상황]` 태그 완전 제거, event text에서 행동 요약으로 대체. LLM이 sceneFrame에 과도하게 반응하는 문제 해결.

2. **`[현재 장면 상태]` 블록**: 대화 상대, 장면 배경, 위치, 직전 행동 등 현재 장면 핵심 맥락을 명시적으로 전달. LLM이 장면 연속성을 유지하는 데 핵심 역할.

3. **장면 연속성 개선**: 9개 ACTION 턴 중 **6건에서 장면 연속성 유지** 확인 (연속된 LOCATION 내 같은 NPC/장소 유지). sceneFrame이 변경된 3건(턴 6, 13, 20)에서도 갑작스러운 장면 점프 없이 자연스럽게 전환.

4. **의도 파서**: 100% 정확도 유지.
