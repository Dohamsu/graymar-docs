## 5. 잔여 이슈 (미해결 + 신규 발견)

### 여전히 미해결

#### [P1] LLM 내면 서술 위반 -- 근본적 한계
- **빈도**: 전체 13턴 중 약 11턴(85%)에서 1회 이상 발생
- **주요 패턴**: "궁금해졌다"(4회), "느꼈다/느끼게 된다"(3회), "직감이 스쳤다"(2회), "다짐했다"(1회)
- **원인 분석**: gpt-4o-mini의 한국어 서술 습관에 "내면 묘사 → 외면 묘사" 전환이 어려움. 특히 서술 마무리에서 "당신은 ~했다"로 끝내려는 경향이 강함.
- **권장 대책**:
  1. 모델 변경 고려 (gpt-4o 또는 Claude 3.5 Sonnet이 지시 추종력 우수)
  2. 후처리(post-processing)에서 금지 패턴 필터링 (정규식 기반)
  3. Few-shot 예시를 2~3개 추가하여 올바른 종결 패턴 학습 유도

#### [P1] LLM 마무리 사족 -- 패턴 변형 우회
- **빈도**: 약 6턴에서 "다음 행동을 고려", "고민하게 된다", "좋을지" 등 발생
- **분석**: 프롬프트에 명시한 금지 패턴을 미세하게 변형하여 우회. "고민하게 된다" → "고려해야 한다", "필요성을 느꼈다" → "느끼게 된다" 등.

#### [P0] LLM 익명 NPC 대사 확장
- **사례**: Run 1 T5 "감독관", Run 1 T2 "젊은 선원"
- **분석**: 프롬프트는 "대사 1~2마디 정도만 허용"이라고 하지만, LLM이 이들을 본격적 대화 상대로 확장. 다음 턴에서 해당 NPC가 서버에 등록되지 않아 소실될 위험.
- **권장 대책**: "익명 배경 인물은 대사를 포함해서는 안 됩니다. 군중 소음이나 간접 전달만 가능합니다." 수준으로 규칙 강화.

### 신규 발견

#### [P1 신규] 현대어 사용 -- "안녕하세요"
- **사례**: Run 1 T0 NPC 대사에서 "안녕하세요" 사용
- **분석**: 프롬프트에 "모든 NPC는 중세 판타지 경어체(~소/~오/~하오)를 기본으로 사용" 명시되어 있으나 LLM이 프롤로그 도입부에서 현대 인사말 사용.
- **권장 대책**: 금지 인사말 목록("안녕하세요", "안녕하십니까", "반갑습니다" 등) 명시. 허용 인사 예시 제공("잘 오셨소", "이쪽이오" 등).

#### [P1 신규] 플레이어 대사 생성
- **사례**: Run 2 T4 "진정하시오, 나는 이 길드 소속이오. 오해를 풀고 싶소"가 큰따옴표 없이 서술에 포함됨
- **분석**: 플레이어의 ACTION 입력 원문을 LLM이 그대로 인용하여 서술에 넣은 것으로 추정. 프롬프트 L83에 "플레이어의 직접 대사를 절대 생성하지 마세요" 명시.
- **빈도**: 2회 Run 중 1회 발생 -- 간헐적이나 몰입 파괴.

#### [P2 신규] LLM 서술 분량 불균형
- **관찰**: T1(313자), T3(387자)은 최소 400자 기준 미달. T6(822자)은 700자 기준 초과.

## 8. 결론

### 전체 개선 평가

**종합 점수: 7.1 → 8.1 (+1.0점)**

서버 엔진 측 수정(P0-1 이벤트 반복 제한, P1-1 PERSUADE 키워드, P2-8 성별 반영, P2-10 Followup 선택지)은 **모두 기대 수준 이상으로 작동**한다. 특히 이벤트 매칭(+2.5)과 선택지 품질(+2.0)의 개선 폭이 크며, 게임 진행의 반복감과 단조로움이 확연히 줄었다.

반면 LLM 프롬프트 측 수정(P0-2 NPC 창작 금지, P1-2 내면 서술 금지, P2-7 사족 제거)은 **부분적 효과만 달성**했다. 프롬프트 설계 자체는 충실하나, gpt-4o-mini의 지시 추종력 한계로 인해 위반이 지속된다.

### 다음 단계 권장 사항

1. **LLM 모델 변경 테스트** (최우선): gpt-4o 또는 Claude 3.5 Sonnet으로 동일 프롬프트 테스트. 지시 추종력 비교 후 결정. 현 프롬프트의 금지 패턴 목록은 충분히 상세하므로 모델 성능만 올라가면 효과가 클 것으로 예상.
2. **후처리 필터** (비용 효율적 대안): LLM 출력에 정규식 기반 필터 적용. "궁금해졌다", "느꼈다", "~해야 한다" 등 반복 패턴을 감지하여 마지막 문장 삭제 또는 교체.
3. **익명 NPC 대사 제한 강화**: 현재 "대사 1~2마디" 허용을 "간접 전달만 가능"으로 격상. 또는 서버에서 LLM에 전달하는 컨텍스트에 "대사 가능 NPC 목록"을 명시적으로 주입.
4. **LOCATION 실전 테스트 확대**: 빈민가(LOC_SLUMS) 미검증. 4 LOCATION 모두 커버하는 테스트 라운드 필요.
5. **에스컬레이션(고집) 시스템 검증**: 같은 LOCATION에서 동일 actionType 3회 반복 시나리오 의도적 테스트 필요.
