# Quality Check — 서술 품질 정밀 점검

플레이테스트 결과 JSON을 기반으로 서술 품질을 항목별로 정밀 점검하고 점수를 매긴다.

## 사용법

| 입력 | 동작 |
|------|------|
| `/quality-check` | 가장 최근 플레이테스트 리포트 자동 탐색 후 점검 |
| `/quality-check playtest-reports/xxx.json` | 특정 리포트 파일 점검 |
| `/quality-check --run` | 10턴 플레이테스트 실행 후 바로 점검 |
| `/quality-check --run 15` | 15턴 실행 후 점검 |
| `/quality-check --run --model qwen/qwen3-235b-a22b-2507` | 특정 모델로 실행 후 점검 |

`$ARGUMENTS`를 파싱:
- 파일 경로가 있으면 해당 파일 사용
- `--run`이 있으면 플레이테스트 먼저 실행
- 숫자가 있으면 턴 수 (기본 10)
- `--model xxx`가 있으면 런타임 모델 전환 후 실행

---

## 절차

### 1단계: 리포트 확보

- `--run` 모드: 서버 확인 → playtest.py 실행 → JSON 저장
- 파일 지정 모드: 해당 파일 로드
- 기본 모드: `playtest-reports/` 에서 가장 최근 .json 파일 사용

### 2단계: 기본 정보 추출

리포트 JSON에서 추출:
- RunID, 프리셋, 턴 수, 검증 결과 (V1~V6)
- 모델명 (DB ai_turn_logs에서 조회)
- 평균 레이턴시, 턴당 비용

### 3단계: 7개 항목 정밀 점검

각 항목을 1~5점으로 평가한다. **반드시 구체적 한국어 서술 인용**과 함께 평가.

---

#### [A] 서술 일관성 (1~5점)

플레이어 행동과 판정 결과가 서술에 정확히 반영되는지.

체크리스트:
- [ ] ACTION 입력과 서술 내용이 일치하는가 (예: "잠입한다" → 잠입 묘사)
- [ ] SUCCESS/PARTIAL/FAIL 판정이 서술 톤에 반영되는가
- [ ] 서버 이벤트(eventId)와 서술 내용이 연관되는가
- [ ] 이전 턴의 결과가 다음 턴에서 자연스럽게 이어지는가

점수 기준:
- 5: 모든 턴에서 행동→판정→서술이 완벽 일치
- 4: 1~2건 불일치
- 3: 3~4건 불일치 또는 판정 미반영
- 2: 절반 이상 불일치
- 1: 서술이 행동/판정과 무관

---

#### [B] NPC 대사 품질 (1~5점)

NPC 대사의 자연스러움, 캐릭터성, 어투 일관성.

체크리스트:
- [ ] 하오체 일관성 (NPC 대사는 "~하오/~이오" 경어체)
- [ ] NPC별 톤 차별화 (회계사 vs 경비병 vs 노부인 등)
- [ ] 대사 내용이 NPC의 role/posture에 부합하는가
- [ ] 동일 NPC가 연속 턴에서 같은 대사를 반복하는가 → 반복 시 감점
- [ ] 대사 길이가 적절한가 (1~2문장 원칙)

점수 기준:
- 5: 모든 NPC 대사가 자연스럽고 캐릭터별 개성 뚜렷
- 4: 대부분 자연스러움, 1~2건 어색
- 3: 톤 차별화 약함 또는 반복 대사 2건+
- 2: 대사 반복 심각 또는 캐릭터 혼동
- 1: 대사 자체가 부자연스러움

---

#### [C] @마커 정확도 (1~5점)

NPC 대사에 붙은 @마커가 실제 화자와 일치하는지.

체크리스트:
- [ ] @마커 NPC와 서술 속 화자가 일치하는가
- [ ] @[당신], @[스치] 같은 오류 마커가 있는가
- [ ] 마커 없는 대사(누락)가 있는가
- [ ] 초상화 URL이 포함되어 있는가
- [ ] NPC 소개 전/후 호칭 전환이 자연스러운가 (unknownAlias → name)

점수 기준:
- 5: 모든 마커 정확 + 초상화 포함
- 4: 1건 누락 또는 불확실
- 3: 2~3건 문제
- 2: 오귀속 또는 자의적 호칭 다수
- 1: 마커 시스템 작동 안 함

---

#### [D] 분위기/감각 묘사 (1~5점)

장소별 분위기, 시각/청각/후각/촉각 묘사의 풍부함.

체크리스트:
- [ ] 장소별 분위기 차이가 느껴지는가 (시장 vs 경비대 vs 항구)
- [ ] 감각 묘사가 2개 이상 감각을 사용하는가 (시각+후각 등)
- [ ] 배경 묘사가 구체적인가 (추상적 "긴장감" vs 구체적 "땀이 이마를 타고")
- [ ] "긴장감/긴장/경계" 같은 단어의 과다 사용이 없는가
- [ ] 다크 정치 RPG 톤에 부합하는가

점수 기준:
- 5: 매 턴 풍부한 감각 묘사 + 장소별 분위기 차별화
- 4: 대부분 양호, 일부 턴에서 묘사 부족
- 3: 감각 1개만 사용하는 턴이 절반 이상
- 2: 묘사가 추상적이고 반복적
- 1: 분위기 묘사 거의 없음

---

#### [E] 판정 반영 차별화 (1~5점)

SUCCESS/PARTIAL/FAIL 판정이 서술에서 명확히 구분되는지.

체크리스트:
- [ ] SUCCESS: 정보 획득, 목표 달성이 명확히 서술되는가
- [ ] PARTIAL: "성공했다" 표현 없이, 구체적 대가 1개가 서술되는가
- [ ] FAIL: 정보 획득 없이, NPC 침묵/거부/적대가 묘사되는가
- [ ] "거의 성공할 뻔했다" 같은 모호한 표현이 FAIL에서 사용되지 않는가
- [ ] FAIL과 PARTIAL의 서술이 서로 구분 가능한가

점수 기준:
- 5: 모든 판정이 서술에서 명확히 차별화
- 4: 대부분 차별화, 1~2건 모호
- 3: FAIL과 PARTIAL이 유사하게 서술됨
- 2: 판정이 서술에 거의 반영 안 됨
- 1: 모든 판정이 동일 패턴으로 서술

---

#### [F] 프리셋 반영 (1~5점)

캐릭터 프리셋(DESERTER/SMUGGLER 등)의 배경이 서술에 반영되는지.

체크리스트:
- [ ] 프리셋 배경이 행동 묘사에 자연스럽게 녹아드는가
- [ ] 프리셋 MANNERISMS가 과다 반복되지 않는가 (예: "절도 있는 움직임" 매 턴)
- [ ] NPC가 플레이어의 배경을 인식하는 반응이 있는가
- [ ] 전투/잠입 등 행동에서 프리셋 특성이 드러나는가

점수 기준:
- 5: 매 턴 자연스러운 프리셋 반영, 과반복 없음
- 4: 대부분 반영, 1~2건 관용구화
- 3: 반영은 되지만 동일 표현 반복 (3회+)
- 2: 간헐적 반영
- 1: 프리셋 배경 무시

---

#### [G] 장기 내구성 (1~5점)

10턴 이후 서술 품질 유지 여부 (10턴 이하 테스트 시 N/A).

체크리스트:
- [ ] 후반부(10턴+)에서 소재 고갈/반복이 심화되는가
- [ ] 동일 소품(종이/창고/숫자)이 5턴 이상 반복되는가
- [ ] 장소 이동 시 소재가 리프레시되는가
- [ ] NPC 대사가 후반부에서 반복되는가
- [ ] 서술 길이가 후반부에서 줄어드는가
- [ ] [MEMORY:] 태그 누출, 턴 번호 노출 등 포맷 오류가 후반부에 집중되는가

점수 기준:
- 5: 15턴까지 품질 유지
- 4: 소폭 반복 증가, 전체적으로 양호
- 3: 반복 패턴 명확히 감지, 소재 재활용
- 2: 후반부 품질 급락
- 1: 5턴 이후부터 반복/복붙

---

### 4단계: 포맷 오류 자동 검출

스크립트로 자동 검출하는 항목:

```python
# 자동 검출 항목
- [MEMORY:], [THREAD], [CHOICES] 태그 누출 (서술에 남아있는지)
- "이전 턴에", "플레이어가", "골드가 지출" 등 메타 서술
- @[당신], @[그대] 등 플레이어 마커
- "턴 N에서" 턴 번호 서술 노출
- 합니다/습니다 혼용 (해라체 위반)
- 동일 문구 3턴 이상 반복
```

### 5단계: 종합 리포트 출력

```
━━ 서술 품질 점검 결과 ━━

모델: {모델명} | 턴: {턴수} | 프리셋: {프리셋}
레이턴시: {평균}초 | 비용: {턴당}원

| 항목 | 점수 | 핵심 |
| [A] 서술 일관성 | X/5 | {한 줄 요약} |
| [B] NPC 대사 | X/5 | {한 줄 요약} |
| [C] @마커 정확도 | X/5 | {한 줄 요약} |
| [D] 분위기/감각 | X/5 | {한 줄 요약} |
| [E] 판정 차별화 | X/5 | {한 줄 요약} |
| [F] 프리셋 반영 | X/5 | {한 줄 요약} |
| [G] 장기 내구성 | X/5 | {한 줄 요약} |
| 종합 | X.X/5 | |

━━ 자동 검출 ━━
태그 누출: X건 | 메타 서술: X건 | 턴번호 노출: X건
해라체 위반: X건 | 반복 문구: {목록}

━━ 좋은 서술 예시 ━━
T{N}: "{인용}" — {왜 좋은지}

━━ 문제 서술 예시 ━━
T{N}: "{인용}" — {문제점}

━━ 개선 권장 ━━
P0: {즉시 수정 필요}
P1: {중요}
P2: {개선}
```

### 6단계: 텔레그램 알림

텔레그램 채널이 활성화되어 있으면 종합 점수와 핵심 발견을 전송한다.

---

## 점수 해석 가이드

| 종합 점수 | 해석 | 권장 조치 |
|-----------|------|----------|
| 4.5+ | 출시 가능 | 유지 |
| 4.0~4.4 | 양호 | 미세 조정 |
| 3.5~3.9 | 수용 가능 | 개선 검토 |
| 3.0~3.4 | 개선 필요 | 파이프라인/프롬프트 수정 |
| 2.5~2.9 | 미흡 | 모델 전환 또는 대규모 수정 |
| 2.0 이하 | 부적합 | 모델 교체 필수 |

## 모델 간 비교 시

동일 프리셋/턴 수로 복수 모델을 테스트한 경우, 항목별 비교표를 추가:

```
━━ 모델 비교 ━━

| 항목 | 모델A | 모델B | 모델C |
| [A] 서술 일관성 | X/5 | X/5 | X/5 |
| ... | ... | ... | ... |
| 종합 | X.X | X.X | X.X |
| 속도 | Xs | Xs | Xs |
| 비용 | X원 | X원 | X원 |
```
